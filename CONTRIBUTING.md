# Guide de contribution

## Branches
- main : branche de production
- dev : intégration
- feature/nom-tache : nouvelles fonctionnalités
- bugfix/nom-bug : corrections

## Workflow
1. Crée une branche depuis dev
2. Travaille + commit clair
3. Push et ouvre une Pull Request vers dev
4. Demande review (au moins 1 personne)
5. Merge après validation

## Conventions de commit
feat: ajout nouvelle fonctionnalité
fix: correction bug
docs: mise à jour documentation
chore: maintenance (dependabot, .gitignore)
